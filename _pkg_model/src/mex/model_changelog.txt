6.0 (8/25/2025)
  - Revised the formulation of multi-channel gain-factor calculation. Previously, the system
    was somewhat convoluted, incorporating an overall MOC weight with a channel-by-channel
    "innervation factor" and a gain-mixing kernel; now, every channel has its own MOC weight
    (so MOC weight is vector-valued) and gain factors are smoothed across CF using a 
    log-frequency Gaussian smoothing kernel. The "innervation factor" system has been removed.
    Parameter `moc_weight` is vector-valued but unused, parameters `moc_beta` and `moc_offset`
    are now vector-valued and apply to the WDR pathway implicitly. The IC pathway is
    effectively disabled for the time being and only the WDR pathway plays a role in 
    efferent responses.

5.4 (11/13/2023)
  - Modified the way that WDR-driven gain control is normalized with respect to the number of
    contributing channels in the underlying simulation. Previously, gain-factor values from
    different channels were combined via multiplication and then normalized by raising the
    final value to the power of 1/n_cf_per_oct. This achieved normalization with respect to 
    the "density" of CFs in the simulation, but changing the width of the WDR projections 
    resulted in changes in the effective strength of WDR-driven gain control. To make it 
    easier to explore the effects of WDR pathway width without confounding it with the 
    overall strength of the WDR pathway, this normalization factor was altered to be
    1/(n_cf_per_oct * moc_width_wdr), which means that WDR-driven gain-factor values
    are now normalized with respect to the number of channels that contributed to the 
    gain factor value. The final calculation can be understood as a geometric mean of all 
    the contributing gain-factor values. If moc_width_wdr is set to zero, we disable this 
    normalization scheme.
5.3 (11/13/2023)
  - Modified Cohc and Cihc to be vector-valued instead of scalar-valued. Each is now a 
    vector of the same shape as the CF vector, so Cohc and Cihc can be different in each
    channel. This change allows simulation of CF-dependent hearing loss.
5.2 (11/06/2023)
  - Refined PLA approximation parameters based on updated optimization scheme. The main 
    change from previous versions is that the stimulus was much more challenging than a pure
    tone and optimization was repeated many times with random initialization values and then
    a single best fit was selected.
5.1 (07/25/2023)
  - Added approximate power-law adaptation. This approximation is based on using many parallel 
    exponential adaptation processes with a wide range of time constants in lieu of the 
    true power-law adaptation. These processes can be implemented efficiently as 
    small recursive filters (single-pole IIR lowpass) and result in a model that is in 
    linear time with respect to input length, whereas the true power-law adaptation 
    superlinear in time. The approximation includes 100 parallel processes each for the slow
    and fast power-law stages, and the range of time constants was fit empirically to 
    match approximate to true output waveforms for the power-law stages in response to 
    brief pure tones at various frequencies (this procedure will be improved in the future
    to give better fits to a wider range of stimuli).

4.2 (04/16/2023)
  - Added arguments to control "fastmode" hacks. In the main model function, 
    powerlaw_include_fast determines whether the fast power-law adaptation is included in
    the model, and powerlaw_len_memory allows the user to limit the maximum memory length
    of the power-law adaptation (in samples). In the wrapper function designed for use
    with mex, we simply expose a fastmode switch that either runs the model without any
    hacks (fastmode==0) or runs the model with the fast power law disabled and a maximum
    power-law memory length of 5000 samples (fastmode==1)
